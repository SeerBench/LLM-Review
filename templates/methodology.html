6 项能力探测的技术细节和评分体系的数学原理

设计原则

只测能力，不测可伪造信号

实际能力不可伪造。回答风格、自报身份、拒答模式都有可能被代理在系统中伪造出来。因此我们弃用所有可被表面模仿的行为指纹，只考核模型解决客观实际问题的能力。

与官方直连 API 评测对照

6 大类能力测试，使用完整版几百项测试题库，通过官方直连 API 评测。评测成绩构成对各主流模型的基线成绩。以此作为对比，直观看到送测模型的能力档位。

确定性自动评分

经过几轮改进，我们放弃了"模糊指标 + 人工对照打分"的模糊评测方式。每道题都有客观答案：解析数学等价、Python Verifier 验每条强约束、模型提供代码直接在 Docker 沙箱跑测试用例。评分标准确定、可复核，不依赖模糊的人工主观评判或者 LLM 当裁判。

优雅降级

任何一类探针失败——Docker 不可用、模型超时、超过一半样本未完成——都会被自动从档位后验里剔除而不污染其他维度，置信度对应下调，报告页明确标注被剔除的类别。

6 项能力探测

以下 6 项探针抽题自月度轮换以抗污染的公开能力题集，每题都有确定性评分器（数学解析、语义匹配、规则验证、沙箱测试用例等）。 每类默认抽 10 题，与公开 leaderboard 同口径采样；下方每张卡片附「公开模型基线」让你直观看到不同档位的预期分数区间。 为避免针对性对抗，我们不对外披露具体题源仓库与子分类标签。

题源：数学竞赛、奥赛、本科高难度题，月度轮换以抗污染。

付费版 · 更完整的检测

付费版将在两条线上同时扩展，做"完整规模能力评测 + 多维度行为佐证"的复合判定。

A

完整规模能力评测

免费版每类抽 10 题（σ ≈ 16pt），对相邻档位判别力有限。付费版按公开榜单同口径完整跑数百题，单次测试就能锁定能力档位，消除小样本带来的方差和噪声。

B

多维度探针套件（13 项）

行为指纹单独使用易被代理 mock，因此从主路径剔除；但作为辅助佐证、与能力分数交叉对比仍有价值。每项一句话原理：

复杂推理5 道精心设计的多步推理题，每题都有自动验证器；最难伪造的能力维度。
魔术拒答某些模型训练时嵌入的特定字符串触发拒答，非本家模型不会触发。
大海捞针在 200K / 500K / 1M 长文本中插入随机验证码，测长上下文检索能力。
混淆抗性用零宽字符等手段包装请求，测模型是否会被绕过或扰乱。
内容门控敏感主题的门控反应曲线，每个 Claude 版本表现略有差异。
身份验证8 种策略让模型透露身份；易被系统提示伪造，仅作辅助证据。
知识边界用不同时点的事件知识探测模型的训练数据截止日期。
数学奥赛5 道 AIME 题 + 1 道经典计数题，整数答案自动判分。
代码补全5 道 HumanEval 边缘任务，subprocess 沙箱直接跑测试用例。
指令遵循6 道带强约束的写作任务，Python verifier 校验每条约束。
代码编程单次生成 Boids / Warp 等可视化 HTML，测一次性创意编程能力。
延迟分析非对称评分——比官方直连更快才可疑（小模型推理速度更快）。
响应元数据检查 response.model 字段与可疑 HTTP 头，最弱信号但零成本。

能力刻度尺（4 档）

当报告含 6 维能力探针数据时，主结论换成能力刻度尺：横轴是 6 个能力类别准确率的等权均值（与公开 leaderboard 相同口径），把测试端点直接放到由公开模型锚定的能力档位上。

阈值取相邻锚点之间的中点，使 GPT-5.5、Opus 4.7、Sonnet 4.6、Haiku 4.5 等公开模型稳定落入对应档位。

能力档位后验

动机：行为指纹（拒答模式、obfuscation 反应、内容门控等）容易被代理 mock 出来 — 一个完全非 Claude 的模型只要不拒答任何敏感内容，就会被错配到 claude-haiku-4-5。新的能力维度判别器走相反的路：用公开可验证的能力题集直接测被测模型的能力档位，再做 Bayesian 后验。

题源：6 个公开能力类别（数学 / 推理 / 语言 / 指令遵循 / 数据分析 / 代码），月度轮换以抗污染；代码维度依赖 Docker 沙箱跑测试用例。

采样：默认每类 10 题，seed 随机生成 + 报告里存 question_id 列表，可重放、不可预先缓存。

评分：使用确定性评分机制（math 用 sympy 解析、reasoning 用语义匹配、IF 用 pure-python verifier、coding 用 Docker 沙箱跑测试用例）。每题输出 5 态结果之一：correct / wrong / refused / parse_error / runtime_error。

后验：每个类别视作独立归一化计算，6 档（Opus 4.7 / Opus 4.6 / Sonnet 4.6 / Sonnet 4.5 / Haiku 4.5 / 低于 Haiku）。计算成绩分布与自我声明的匹配程度。

已知校准问题

  • •公开模型基线按 thinking-on 校准；我们 send_message 不传 thinking 参数 → 绝对分被 1-2 档下移，但相对排序仍可比较。
  • •免费快速测试 60 题 σ ≈ 16pt，单次评测结果可能存在波动；付费完整测试（数百题）置信度更高。
  • •代码维度需 host 安装 Docker；未安装时维度自动剔除，confidence=0.1。

对抗与防御

本工具一旦被中转站知晓，其经营者就可能采取针对性的对抗手段以掩盖降级行为。我们持续投入资源识别并削弱这些手段，让评测结果对最终用户尽可能真实。

中转站可能的做法

  • •抓取并缓存测试题库，对已知题目预生成正确答案以骗过判分。
  • •记录评测方服务器的固定出口 IP，仅对该 IP 提供完整旗舰模型，对普通用户继续走降级路径。
  • •识别请求里的特征（固定 prompt 模板、固定 user-agent、固定批量节奏）后切换到"演戏模式"。

我们的针对性措施

  • •题库随机抽取：每次测试用随机种子从大题集里抽题，单题被重复命中的概率极低。
  • •题库定期更新：以月为周期轮换题源，已外泄或被批量抓取的题目会及时退出在用题集。
  • •随机 IP 保护：测试请求经住宅代理池随机出口，让中转方看到的来源 IP 与普通终端用户难以区分。

我们无意宣称防御是完美的——这是一场长期博弈。但尽最大可能为用户提供有效、可信的评测数据，是慧眼鉴真的价值所在。

已知局限性

  • •小样本方差：免费版每类抽 10 题，单次测试 σ 较大，建议多次取平均判断；付费版将开放完整题库（每类数百题），单次测试即可锁定能力档位。
  • •题库污染风险：若被测模型在训练时已经见过题集中的部分题目，分数会虚高。我们以月为周期轮换题源、用随机种子抽样以降低命中概率，但完全杜绝训练污染并不现实。
  • •基线对照差异：公开模型的基线分数取自其官方"开启思考"配置；本工具为兼容更广的代理协议默认采用无 thinking 模式，被测端点的绝对分通常较公开榜偏低 1-2 档，相邻档位之间的相对排序仍可比较。
